Fast in-memory cache

MCache 现在不仅是一个高性能 Go 内存缓存库，也可以按集群模式运行，支持 replicated writes、成员管理、健康探针、Prometheus 指标和故障恢复验证。

Get Started See Cluster Mode
main.go
# start node-1 in raft mode
MCACHE_CLUSTER_MODE=raft \
MCACHE_NODE_ID=node-1 \
MCACHE_HTTP_ADDR=0.0.0.0:8081 \
MCACHE_ADVERTISE_ADDR=http://127.0.0.1:8081 \
MCACHE_RAFT_BIND_ADDR=127.0.0.1:7001 \
MCACHE_RAFT_ADVERTISE_ADDR=127.0.0.1:7001 \
MCACHE_RAFT_BOOTSTRAP=true \
go run ./pkg

# inspect readiness and metrics
curl http://127.0.0.1:8081/readyz
curl http://127.0.0.1:8081/metrics

Features

⚡

极低延迟

基于 sync.Map 实现无锁读路径，单线程 Get 吞吐量达 18M ops/sec，并发场景下超过 52M ops/sec。

🌲

多级 Key 路径

以 / 分隔的层级路径作为 key，通过前缀树维护层级关系，支持按路径前缀批量查询子节点。

🗳

Raft 集群模式

支持单机模式和 raft 模式切换，写请求通过日志复制达成一致，具备 leader 选举、故障切换和快照恢复能力。

🔌

双模式集成

支持以 Go module 直接嵌入宿主程序（零网络开销），也可作为独立进程通过 HTTP/gRPC 对外提供服务。

📈

可观测性

内置 /livez、/readyz、/metrics 和 /v1/cluster/diagnostics，方便接入 Prometheus 和容器编排探针。

🧪

恢复验证

仓库内置 smoke、quorum 和 rolling restart 三类 e2e 脚本，用于验证 failover、成员恢复和多数派写入行为。

Quick Start

embed.go
import (
    "github.com/mcache-team/mcache/pkg/mcache"
    "github.com/mcache-team/mcache/pkg/apis/v1/item"
)

c := mcache.New()

// Insert with 5-minute TTL
c.Insert("user/session/token", "abc123", item.WithTTL(5*time.Minute))

// Get exact key
it, err := c.Get("user/session/token")

// Update data
c.Update("user/session/token", "xyz789")

// List children under a prefix
items, _ := c.ListByPrefix("user/session")

// Delete
c.Delete("user/session/token")
http-api.sh
# Start the server
docker run -p 8080:8080 ghcr.io/mcache-team/mcache

# Insert
curl -X PUT http://localhost:8080/v1/data \
  -H 'Content-Type: application/json' \
  -d '{"prefix":"user/name","data":"alice"}'

# Get
curl http://localhost:8080/v1/data/user%2Fname

# Update
curl -X POST http://localhost:8080/v1/data/user%2Fname \
  -d '{"data":"bob"}'

# List by prefix
curl "http://localhost:8080/v1/data/listByPrefix?prefix=user"

# Delete
curl -X DELETE http://localhost:8080/v1/data/user%2Fname

Cluster Mode

从单机内存缓存到 raft-backed replicated service，保留简洁接口，同时补齐真实运行时能力。

👑

Leader-aware Writes

写请求统一经过 cluster node 提交；leader 负责追加 raft 日志，follower 会返回 307 Temporary Redirect。

🧩

Dynamic Membership

支持通过 /v1/cluster/nodes 动态添加和移除 voter 节点，不需要每次都靠静态 peer 配置重启整套集群。

💽

Snapshot & Restore

raft 日志配合文件快照和本地状态机恢复，在节点重启、追日志和 leader 切换后保持数据状态可延续。

Architecture

现在的核心已经从单纯的 handler + memory store，收口成可被 raft 驱动的统一状态机。

Clients
HTTP :8080 gRPC :9090 Go Module /metrics
↓
Cluster Node
Single / Raft Leader Redirect Membership Diagnostics
↓
State Machine
Items + Prefix Index Command Apply Snapshot / Restore Telemetry

Servers

对外提供 REST、health probes、metrics 和 cluster 管理入口。HTTP 写请求统一汇聚到 cluster node，再决定本地执行还是 raft 复制。

Cluster

封装 single / raft 两种运行模式，对外暴露统一的 submit、成员管理、ready 检查和 diagnostics 接口，便于服务层无差别调用。

State Machine

KV 数据和 prefix index 统一维护在同一个状态机里，通过 command 驱动 apply，便于快照、恢复和后续继续扩展分片或更多复制策略。

Ops & Recovery

不仅能跑起来，也尽量把“怎么确认它是对的”一起做进仓库里。

🩺

Health Probes

/livez 提供存活检查，/readyz 和 /healthz 会基于当前 raft 角色和 leader 状态给出 ready 结果。

📊

Prometheus Metrics

输出缓存条目数、成员数、raft 索引、写请求计数，以及标准 histogram 形式的写延迟指标，便于直接接 Prometheus。

🔁

Recovery E2E

内置 smoke、quorum、rolling restart 三类脚本，覆盖 leader failover、少数派拒写、旧 leader 恢复和逐节点重启验证。

ops.sh
# local raft smoke test
make e2e-raft

# quorum behavior test
make e2e-raft-quorum

# rolling restart recovery test
make e2e-raft-rolling
