Fast in-memory cache

MCache 是一个高性能 Go 内存缓存库，支持多级 key 路径、TTL 过期、HTTP/gRPC 服务模式，以及定时落盘持久化。

Get Started View on GitHub
main.go
import "github.com/mcache-team/mcache/pkg/mcache"

c := mcache.New()

// insert with TTL
c.Insert("user/profile/name", "alice", item.WithTTL(time.Minute))

// get by exact prefix
it, _ := c.Get("user/profile/name")

// list all children under a path
items, _ := c.ListByPrefix("user/profile")

Features

⚡

极低延迟

基于 sync.Map 实现无锁读路径，单线程 Get 吞吐量达 18M ops/sec，并发场景下超过 52M ops/sec。

🌲

多级 Key 路径

以 / 分隔的层级路径作为 key，通过前缀树维护层级关系，支持按路径前缀批量查询子节点。

⏱

TTL 过期

每个缓存条目可独立设置存活时长，采用惰性删除策略，读取时自动检测并清理过期数据。

🔌

双模式集成

支持以 Go module 直接嵌入宿主程序（零网络开销），也可作为独立进程通过 HTTP/gRPC 对外提供服务。

💾

定时落盘

通过环境变量 PERSIST_DIR 启用定时快照，服务重启后自动从磁盘恢复数据，过期条目自动跳过。

🔒

并发安全

Insert 使用 LoadOrStore 原子操作保证唯一性，prefixList 读写通过 RWMutex 保护，并发写入安全。

Quick Start

embed.go
import (
    "github.com/mcache-team/mcache/pkg/mcache"
    "github.com/mcache-team/mcache/pkg/apis/v1/item"
)

c := mcache.New()

// Insert with 5-minute TTL
c.Insert("user/session/token", "abc123", item.WithTTL(5*time.Minute))

// Get exact key
it, err := c.Get("user/session/token")

// Update data
c.Update("user/session/token", "xyz789")

// List children under a prefix
items, _ := c.ListByPrefix("user/session")

// Delete
c.Delete("user/session/token")
http-api.sh
# Start the server
docker run -p 8080:8080 ghcr.io/mcache-team/mcache

# Insert
curl -X PUT http://localhost:8080/v1/data \
  -H 'Content-Type: application/json' \
  -d '{"prefix":"user/name","data":"alice"}'

# Get
curl http://localhost:8080/v1/data/user%2Fname

# Update
curl -X POST http://localhost:8080/v1/data/user%2Fname \
  -d '{"data":"bob"}'

# List by prefix
curl "http://localhost:8080/v1/data/listByPrefix?prefix=user"

# Delete
curl -X DELETE http://localhost:8080/v1/data/user%2Fname
grpc-client.go
import grpcclient "github.com/mcache-team/mcache-sdk-go/grpc"

// Connect to gRPC server (port 9090)
c, err := grpcclient.New("localhost:9090")
defer c.Close()

c.Insert(ctx, "user/name", "alice", mcache.InsertOption{
    TTL: 10 * time.Minute,
})

it, _ := c.Get(ctx, "user/name")
fmt.Println(it.Data) // alice
persist.sh
# Enable persistence: flush every 30s to /data
PERSIST_DIR=/data PERSIST_INTERVAL=30s ./mcache

# On restart, data is automatically restored from:
#   /data/mcache-snapshot.json
# Expired entries are skipped during restore.

# With Docker
docker run \
  -e PERSIST_DIR=/data \
  -e PERSIST_INTERVAL=1m \
  -v $(pwd)/data:/data \
  -p 8080:8080 \
  ghcr.io/mcache-team/mcache

Architecture

三层架构，职责清晰，存储后端可替换。

Clients
HTTP :8080 gRPC :9090 Go Module
↓
Handlers
PrefixTree InsertNode RemoveNode ListNode
↓
Storage
sync.Map prefixList Snapshot / Restore

Servers

对外提供多种协议入口：HTTP REST（Gin）、gRPC（port 9090）、Go module 直接调用。所有入口共享同一套 Handler 和 Storage 实例。

Handlers

维护内存中的前缀树（PrefixTree），负责路径解析、节点插入/删除/查询，并将实际数据读写委托给 Storage 层。

Storage

当前实现为内存存储（Memory），使用 sync.Map 保证并发安全，实现 Storage 接口，未来可无缝替换为 Redis 等后端。
